# Frontend Workspace

هذا المجلد هو المصدر الفعلي والوحيد للواجهة الحالية.

## البنية

- `index.html`
  نقطة الدخول التي يخدمها Express.

- `js/core/`
  طبقة التشغيل المشتركة: `api`, `auth`, `router`, `runtime`, `store`, `ui`.

- `js/auth/`
  شاشات وهوكات الدخول والجلسة.

- `js/admin/`
  شاشات المستخدم والإعدادات الشخصية وإعدادات الإدارة.

- `js/shell/`
  هيكل التطبيق: الملاحة، الحماية، الإجراءات السريعة، سجل المسارات، وتوزيع التجربة حسب الدور.

- `js/flows/`
  تدفقات العمل العملية، وحاليًا أهمها:
  - `commercial/` لعروض الأسعار والفواتير والسندات
  - `operations/` للفرص والمشاريع والموظفين والعقود
  - `procurement/` لطلبات ومرتجعات الشراء
  - `inventory/` للأصناف والمستودعات والجرد
  - `finance/` للأستاذ العام وكشف الحساب ومعالج الإقفال

- `js/insight/`
  واجهات المتابعة والتحليل مثل لوحة القيادة والتقارير والتحليلات.

- `js/modules/`
  وحدات قديمة متبقية لحدود أعمال أخرى، ولم تعد تحتوي `operations` أو `settings` أو `reports`.

- `styles/`
  طبقات التصميم: tokens, base, layout, components.

## قواعد العمل

1. أي تطوير جديد للواجهة يتم داخل هذا المجلد فقط.
2. لا تُنشأ `node_modules` داخل مجلدات فرعية.
3. الوحدات المشتركة تبقى في `js/core`، والدخول في `js/auth`، والإدارة الشخصية في `js/admin`، وهيكل التطبيق في `js/shell`، والتدفقات في `js/flows`.
4. أي فصل جديد بين الفرونت والباك يعتمد على عقود API واضحة، وليس على نسخ شاشات أو builds منفصلة.
